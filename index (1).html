FraudShield ML Engine

An end-to-end AI-powered fraud detection system built with supervised machine learning, feature engineering, and real-time transaction scoring.

JavaScript ML Random Forest Logistic Regression SVM Approximation AUC-ROC Feature Engineering Ensemble Learning Anomaly Detection GitHub Pages

🌳 Random Forest

Ensemble of 100 decision trees trained on transaction features. Majority voting across trees produces robust fraud probability scores with high precision on imbalanced data.

📐 Logistic Regression

Linear classifier with L2 regularization. Fast, interpretable baseline that models the log-odds of fraud as a weighted sum of engineered features. Great for probability calibration.

⚡ Ensemble Voting

Weighted average of RF (60%) and LR (40%) scores. Reduces variance from individual models and consistently outperforms either model alone on AUC-ROC.

🔬 Feature Engineering

Amount-to-average ratio, velocity (txn/hour), time-of-day risk, geographic anomaly score, merchant novelty flag, and card-not-present indicator.

🎯 Anomaly Detection

Unsupervised z-score based anomaly flagging on transaction amount, frequency, and merchant category. Catches novel fraud patterns not seen in training data.

📊 Evaluation Metrics

Precision, Recall, F1-Score, AUC-ROC. Optimized threshold at 0.42 to maximize recall (catch more fraud) while keeping FPR low for customer experience.
